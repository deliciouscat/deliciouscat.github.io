> **시리즈**: 05. Agent 실행 흐름 설계 — 2/4

## 개요

Pydantic-AI는 그래프(Node/Edge) 기반 분기 대신, 도구 반환값에 따라 LLM이 자연스럽게 분기를 결정하는 방식을 사용한다. 이 패턴으로 복잡한 흐름을 설계하는 방법을 다룬다.

---

## 세부 목차

### 1. Graph 기반 분기 vs Tool 기반 분기
- LangGraph의 Node/Edge/State 패턴
- Pydantic-AI의 "도구 반환값 → LLM 판단 → 다음 도구 호출" 패턴
- 각 방식의 장단점 비교

### 2. 도구 반환값을 통한 흐름 제어
- 도구가 반환하는 문자열이 LLM의 다음 행동을 유도
- "성공" / "실패" / "추가 정보 필요" 분기 설계
- 도구 반환값에 힌트를 포함시키는 전략

### 3. 조건부 도구 등록
- 상황에 따라 다른 도구 세트를 Agent에 제공
- `Agent(tools=[...])` 동적 구성
- 의존성 기반 도구 필터링

### 4. 순차 도구 호출 패턴
- 데이터 수집 → 분석 → 결과 반환 파이프라인
- 도구 간 데이터 전달 (도구 반환값 → LLM → 다음 도구 인자)
- 중간 결과 축적 패턴

### 5. 재귀적 도구 호출
- 도구가 자기 자신의 재호출을 유도하는 패턴
- 트리 탐색, 페이지네이션 처리
- 무한 루프 방지를 위한 안전장치

### 6. 실전 예제
- 고객 문의 분류 → 전문 처리 흐름
- 데이터 검증 → 보정 → 최종 확인 파이프라인
